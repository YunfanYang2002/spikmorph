# SpikMorph 更新日志

本文件按 Asia/Shanghai（UTC+08:00）时间记录项目变更，最新记录位于最前。

## 2026-07-15

### MuJoCo static root-z clearance sweep

- 新增独立静态 root-z 扫描工具，对每个候选执行 reset、model-default state 注入和 `mj_forward`，不调用 `mj_step`，用于定位首个无 robot-ground contact 的高度。
- 输出粗扫、细化边界、penetration/contact 明细及 safety-margin 推荐高度；无 clear 候选时明确返回非零并建议扩大扫描上限。
- 增加纯 Python 测试，覆盖零 physics step、候选隔离、ground/self-contact 过滤、细化边界、严格 JSON 和无 clear 失败语义。

## 2026-07-14

### MuJoCo first ground contact window

- transition dump 增加可选的首次 robot-ground contact physics-substep 窗口，复用正式 `env.step()` 的 action、reward 与 termination 路径，并记录原始 root、torso、joint、actuator 和 contact force 数据。
- ground geom 通过 compiled runtime ID 唯一识别；无接触、窗口不完整或非有限 JSON 数据均明确失败，默认 transition dump 行为保持不变。
- 增加纯 Python 测试，覆盖 ground/self-contact 区分、首触发 ring buffer、penetration depth、6D force、严格 JSON 与无 auto-reset 约束。

### MuJoCo runtime dynamics metadata

- transition dump 增加按 runtime joint、actuator 和 body name 组织的 compiled dynamics metadata，供后续跨后端按名称对齐比较。
- joint 参数通过 compiled qpos/DOF address 读取，body COM 与 inertia-frame 语义显式记录；缺失或非有限字段写为 `not_available`。
- 本地通过语法编译和 17 项不依赖 MuJoCo runtime 的纯 Python 测试；未调整任何物理参数。

### MuJoCo physics substep transition 记录

- transition dump 增加可选 physics-substep 记录模式，在不改变 `frame_skip` 和正式仿真语义的前提下记录每次 `sim.step()` 后的状态与接触。
- metadata 增加首次接触 physics step 及连续无接触前缀长度，默认关闭时继续保留原 control-step 输出行为。
- 本地通过语法编译和不依赖 MuJoCo runtime 的纯 Python 索引、前缀及兼容性测试；未运行 MuJoCo。

## 2026-07-13

### MuJoCo transition dump 诊断工具

- 新增无渲染、单环境的 transition dump 工具，用 canonical state 和 zero/单关节正负 ctrl 记录可供 Isaac 对照的 JSON 数据。
- 工具按编译模型地址解析 root、joint、actuator、DOF 与接触信息，不修改训练环境、训练代码或 XML。
- 增加目标 joint 初始位置选项，可在保留其他 joint compiled model-default 的同时，仅将目标 limited scalar joint 设置到 compiled range midpoint。
- 本地仅执行语法编译和不依赖 MuJoCo runtime 的纯 Python 检查；MuJoCo 运行留待服务器环境验证。

## 2026-06-23 14:29:33 +08:00

### SNN 5 层 FFN512 实验结果核对

- 确认 `events.out.tfevents.1782096688.batc-PowerEdge-T640.32546.0` 为 `snn_5layer_ffn512_10m` 对应结果文件。
- TensorBoard scalar 仅包含 `Reward`，共 117 个点，step 范围为 491,520 到 9,994,240。
- 最后 reward 与最高 reward 均为 302.61，最后 10 个点均值约 296.50。
- 与此前误认为 FFN512 的文件相比，该 FFN512 结果低于 5 层 FFN256 的 325.53/330.41，说明单纯扩大 FFN 到 512 在当前 SNN 设置下没有带来收益。

## 2026-06-18 16:16:10 +08:00

## 2026-06-18 11:24:03 +08:00

### 训练过程与资源收尾修复

- 将 `PPO.MAX_STATE_ACTION_PAIRS` 和 `PPO.EARLY_EXIT_STATE_ACTION_PAIRS` 的默认值由浮点数改为整数，命令行可直接使用 `16384` 等整数覆盖。
- 在主进程 PPO 训练循环中加入 `tqdm` 进度条，显示 update、已完成环境步数、FPS 和可用时的最近回报。
- 短训尚未产生完整 episode 时不再对空数组求均值，消除 `Mean of empty slice` 和 `invalid value` 警告。
- 训练结束或异常退出时显式关闭向量环境和 TensorBoard writer，避免 `SubprocVecEnv.__del__` 阶段出现 `BrokenPipeError`。
- 增加 `tqdm` 运行依赖。
