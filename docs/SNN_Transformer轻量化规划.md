# SpikMorph SNN Transformer 轻量化规划

## 1. 背景与结论

当前默认 Actor-Critic 模型约有 331 万参数，其中 Actor 和 Critic 分别持有一套完整的 5 层编码器。现有 `SpikingTransformerEncoderLayerResidual` 虽然将中间激活改为脉冲神经元，并采用 `K ⊙ V` 聚合，但仍保留：

- 4 个 `D × D` 注意力投影：Q、K、V、输出投影；
- 两层 `D → 1024 → D` FFN；
- 5 个互不共享权重的编码层；
- Actor 与 Critic 两套互不共享的完整编码器。

因此，当前 SNN 版本与普通 Transformer 的参数量基本相同。SNN 本身主要提供稀疏事件计算和潜在能效优势，并不会自动减少参数；参数压缩必须来自结构设计。

本轮优化目标是：在保持 PPO、环境接口和节点级动作解码协议不变的前提下，将模型参数量首先压缩到 100 万以内，主方案争取达到 25 万左右，同时通过分阶段实验控制性能退化。

## 2. 当前参数构成

默认配置为 `D=128`、`FFN=1024`、`N=5`、双头注意力。单个编码层参数量约为：

| 模块 | 单层参数量 | 占单层比例 |
| --- | ---: | ---: |
| Q/K/V/输出投影 | 66,048 | 20.0% |
| FFN | 263,296 | 79.8% |
| LayerNorm | 512 | 0.2% |
| 合计 | 329,856 | 100% |

Actor 和 Critic 的 10 个编码层合计约 330 万参数，已经占模型参数的 99% 左右。输入 embedding、位置编码和动作/价值解码头都不是主要矛盾。

这意味着优化优先级应当是：

1. 缩小 FFN 扩张率；
2. 减少物理层数；
3. 共享 Actor/Critic 的形态编码；
4. 再缩小 embedding 维度或进行跨层权重共享；
5. 最后才考虑低秩 Q/K/V 等复杂方案。

## 3. 目标与约束

### 3.1 参数目标

- 保守版本：不超过 100 万参数，较当前减少至少 70%；
- 推荐版本：约 23 万参数，较当前减少约 93%；
- 激进版本：不超过 10 万参数，较当前减少约 97%。

以上为默认无地形 MLP 配置的估算。`csr.yaml` 和 `obstacle.yaml` 的外部观测 MLP 会增加少量参数，但不改变数量级。

### 3.2 性能目标

模型压缩不能只比较最终最好回报。建议同时约束：

- 相同环境步数下，最终平均回报达到原模型的 90% 以上；
- 达到基准回报阈值所需环境步数不超过原模型的 1.25 倍；
- 至少 3 个随机种子，报告均值和标准差；
- 不出现 KL、梯度范数或 value loss 持续异常；
- 同时报告参数量、实际前向延迟、训练吞吐和脉冲发放率。

“参数更少”和“SNN 更节能”是两个独立结论。若没有部署硬件上的能耗测量，不应只凭脉冲稀疏度声称实际节能。

### 3.3 工程约束

- 保持现有 observation、padding mask 和 action mask 语义；
- 普通 Transformer 保留为可复现实验基线；
- 新结构通过配置启用，不覆盖现有 checkpoint 行为；
- 参数统计必须区分总参数、可训练参数和共享参数；
- 每个结构变化都应能单独消融。

## 4. 分阶段结构方案

### 阶段 A：低风险瘦身

配置建议：

```yaml
MODEL:
  LIMB_EMBED_SIZE: 128
  TRANSFORMER:
    DIM_FEEDFORWARD: 256
    NLAYERS: 3
```

保持 Actor/Critic 独立，只将 FFN 从 1024 缩至 256，并将层数从 5 缩至 3。

预计参数量约 81 万，较当前减少约 75.5%。这是首个必须完成的基线，因为改动只涉及现有配置，能够快速判断当前任务是否真的需要大 FFN 和 5 层深度。

### 阶段 B：共享形态编码器

将 Actor 和 Critic 改为：

```text
limb observation
       │
 shared spiking encoder
       ├── actor decoder → action mean
       └── critic decoder → state value
```

共享输入 embedding、位置编码和 SNN 编码器，Actor/Critic 仅保留独立的归一化或轻量 adapter 与输出头。使用 `D=128`、`FFN=256`、3 层时，预计约 41 万参数，较当前减少约 87.8%。

主要风险是策略梯度和价值损失对共享特征产生冲突。应采取以下措施：

- Actor/Critic 使用独立的末端 LayerNorm；
- 为两个分支各保留一个小型 bottleneck adapter；
- 记录共享编码器上 actor loss 与 critic loss 的梯度余弦相似度；
- 若冲突明显，再比较“共享前两层、末层独立”的折中结构。

不建议一开始采用完全独立双编码器，因为这会直接浪费一半参数；是否需要独立应由消融结果证明。

### 阶段 C：推荐主模型

推荐配置：

```yaml
MODEL:
  LIMB_EMBED_SIZE: 96
  TRANSFORMER:
    NHEAD: 2
    DIM_FEEDFORWARD: 192
    NLAYERS: 3
  SHARED_ACTOR_CRITIC_ENCODER: true
```

该结构保持每个注意力头 48 维，FFN 扩张率为 2。默认任务下预计约 23 万参数，较当前减少约 93%。

选择 96 维而不是直接降到 64 维，原因是当前每个 limb 的输入已经有 52 个特征，96 维仍保留合理的特征展开空间；同时两头注意力可以整除，不需要修改 mask 或节点协议。

这是建议作为论文或项目主结果的模型：压缩幅度足够显著，结构仍然可解释，风险也低于极端低秩或二值权重方案。

### 阶段 D：跨层权重共享

只保留一个物理 SNN 编码层，在前向中循环执行 3 次；每次循环保持独立膜状态演化，必要时加入可学习的 step embedding 或每步独立 LayerNorm。

在 `D=96`、`FFN=192` 下，预计参数量约 8.1 万，较当前减少约 97.6%。该方案减少参数但不会同比减少计算量。

风险包括：

- 表达能力受限，所有深度使用同一变换；
- SNN 时间步与层循环的状态边界容易混淆；
- checkpoint 和 attention map 的层语义需要重新定义。

因此它应作为激进消融，不应先于阶段 B/C 实施。

### 阶段 E：可选的进一步压缩

只有阶段 C 仍无法达到目标时，才考虑：

- 合并 K/V 投影或共享 Q/K 权重；
- 对 Q/K/V/输出投影使用低秩分解；
- 将 FFN 替换为门控的低秩脉冲 MLP；
- 对 decoder 和外部观测 MLP 进行权重共享；
- 训练后量化到 INT8 或更低位宽。

低秩与量化应与“参数个数”分开报告：低秩减少参数个数，量化主要减少存储字节数。

## 5. 代码改造规划

### 5.1 配置层

在 `metamorph/config.py` 增加：

```python
MODEL.SHARED_ACTOR_CRITIC_ENCODER = False
MODEL.TRANSFORMER.SHARE_LAYER_WEIGHTS = False
MODEL.TRANSFORMER.ACTOR_CRITIC_ADAPTER_DIM = 0
```

现有 `LIMB_EMBED_SIZE`、`DIM_FEEDFORWARD` 和 `NLAYERS` 已足以支持阶段 A/C，不需要引入重复配置。

### 5.2 模型层

建议将目前同时负责 embedding、encoder、外部特征和 decoder 的 `TransformerModel` 拆分为：

- `MorphologyEncoder`：limb embedding、位置编码、SNN/ANN encoder；
- `TaskFeatureEncoder`：hfield 等外部观测；
- `ActorHead`：节点动作均值；
- `CriticHead`：节点价值并聚合；
- `ActorCritic`：组织共享或独立编码路径。

这样共享发生在明确的形态表征边界，不需要通过两个 `TransformerModel` 对象隐式绑权重。

### 5.3 参数统计与测试

新增一个独立脚本，至少输出：

- 全模型总参数与可训练参数；
- encoder、actor head、critic head、外部观测编码器的参数量；
- 共享张量只统计一次；
- 配置摘要和理论压缩率。

新增单元测试：

- Actor/Critic 输出 shape 与现状一致；
- padding limb 不影响有效 limb 输出；
- 共享模式下编码器参数对象确实相同；
- 一次前向、反向后梯度均为有限值；
- LIF/PLIF 以及 torch/cupy fallback 路径可运行；
- 参数量不超过各版本预算。

## 6. 实验矩阵

建议按以下顺序执行，每次只引入一个主要变量：

| 编号 | 编码器 | D | FFN | 层数 | A/C 共享 | 层共享 | 预计参数 | 目的 |
| --- | --- | ---: | ---: | ---: | --- | --- | ---: | --- |
| B0 | ANN Transformer | 128 | 1024 | 5 | 否 | 否 | 331 万 | 原始基线 |
| S0 | 当前 SNN | 128 | 1024 | 5 | 否 | 否 | 331 万 | 验证单纯脉冲化效果 |
| S1 | SNN | 128 | 256 | 3 | 否 | 否 | 81 万 | 低风险压缩 |
| S2 | SNN | 128 | 256 | 3 | 是 | 否 | 41 万 | 验证 A/C 共享 |
| S3 | SNN | 96 | 192 | 3 | 是 | 否 | 23 万 | 推荐主模型 |
| S4 | SNN | 64 | 128 | 3 | 是 | 否 | 约 11 万 | 宽度下限消融 |
| S5 | SNN | 96 | 192 | 3 次循环 | 是 | 是 | 约 8.1 万 | 极限参数压缩 |

每个 SNN 结构至少比较 `T=2` 和 `T=4`。不要同时扩大 `T` 来补偿所有压缩损失，否则很难判断收益来自结构还是更高计算预算。

## 7. 验收指标与停止条件

阶段 A 验收：

- 参数量小于 100 万；
- 训练稳定；
- 3 seeds 平均回报达到 B0 的 90%。

阶段 B/C 验收：

- 参数量分别小于 45 万和 25 万；
- S3 平均回报达到 S0 的 90%，或在同等参数预算下显著超过 ANN 小模型；
- 提供脉冲发放率和前向延迟，避免只报告参数量。

出现以下情况时停止继续压缩，并回退到上一个版本：

- 多个 seed 都无法达到原模型 80% 回报；
- value loss 长期发散且 adapter/损失权重调整无效；
- 参数下降但实际训练和推理成本因时间步展开明显恶化；
- 稀疏率很低，SNN 路径退化为高成本稠密计算。

## 8. 推荐实施顺序

1. 增加参数统计脚本，固定 B0/S0 的可复现数字；
2. 仅通过配置完成 S1，并训练验证；
3. 重构共享 `MorphologyEncoder`，完成 S2；
4. 加入 96 维配置并完成 S3；
5. 完成 S4 宽度消融；
6. 只有 S3 结果稳定后，再实现 S5 跨层共享；
7. 最后考虑低秩和量化，不将其与核心结构改造混在同一轮实验中。

## 9. 预期结果

本规划的主线不是简单把 Transformer 政名为 SNN，而是建立一个参数效率明确、可消融的共享脉冲形态编码器。推荐版本 S3 的目标是把模型从约 331 万参数降至约 23 万，在参数减少约 93% 的情况下维持可接受的控制性能；若性能损失过大，则以 S2 的约 41 万参数版本作为稳健回退方案。
